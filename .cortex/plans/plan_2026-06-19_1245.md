## Context

Currently the app renders a Kanban-style board with 3 columns (To Do, In Progress, Done). The server already returns issues grouped by status category with all the data we need. We'll redesign the client to show an **analytics/stats dashboard** instead.

## Implementation Steps

1. **Add `chart.js` + `react-chartjs-2`** to client dependencies — lightweight charting library, no heavy bundle.

2. **Rewrite `server/routes/jira.js`** — Change the response to return richer stats:
   - Status breakdown (count per status name, not just category)
   - Priority breakdown (count per priority)
   - Assignee breakdown (count per person)
   - Story points totals (done vs remaining)
   - Keep sprint metadata (start/end dates for days-remaining calc)

3. **Replace client components** — Remove `BoardColumn.jsx` and `IssueCard.jsx`. Create:
   - `components/StatCard.jsx` — Single metric card (number + label)
   - `components/PieChart.jsx` — Reusable pie chart wrapper (for status & priority)
   - `components/AssigneeBar.jsx` — Horizontal bar showing work per assignee
   - `components/SprintCountdown.jsx` — Days remaining + progress bar

4. **Rewrite `App.jsx`** — New layout with stat cards row, two pie charts side by side, and assignee section below.

5. **Rewrite `App.css`** — Dashboard grid layout with cards, chart containers, and responsive design.
