## Implementation Steps

1. **Root setup** — Create root `package.json` with `concurrently` to run server + client together via `npm run dev`.

2. **Server** — Express app with:
   - `GET /api/sprint/active` — Fetches the active sprint for a given board
   - `GET /api/sprint/:sprintId/issues` — Fetches all issues in a sprint, grouped by status category
   - JIRA API calls use `axios` with Basic Auth (base64 of `email:apiToken`)
   - Reads credentials from `.env` (`JIRA_BASE_URL`, `JIRA_EMAIL`, `JIRA_API_TOKEN`, `JIRA_BOARD_ID`)

3. **Client** — React app (Vite) with:
   - **SprintHeader** — Shows sprint name, goal, start/end dates, progress bar
   - **BoardColumn** — Kanban column for each status category (To Do, In Progress, Done)
   - **IssueCard** — Shows issue key, summary, assignee avatar, priority icon, and story points
   - Fetches data from the Express backend on mount
   - CSS Grid layout for the 3-column board

4. **Environment config** — `.env.example` with placeholders for the user to fill in their JIRA Cloud URL, email, API token, and board ID.

## How to Run

```bash
# 1. Install all dependencies
npm install && cd server && npm install && cd ../client && npm install && cd ..

# 2. Copy .env.example to .env and fill in JIRA credentials
cp .env.example .env

# 3. Start both server + client
npm run dev
```

Server runs on port 5000, client on port 5173, client proxied to server for `/api` calls.
